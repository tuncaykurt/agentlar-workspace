# Skill: Episode Planning

## Purpose
Transform a research topic into a structured episode plan ready for production.

## Serves Goals
- Grow downloads (compelling titles and descriptions drive clicks)
- Retain listeners (clear structure keeps people listening)
- Drive conversions (strategic CTA placement)

## Inputs
- Research output from `outputs/YYYY-MM-DD_podcast_research.md`
- `knowledge/AUDIENCE.md` — listener language and pain points
- `knowledge/BRAND.md` — voice and tone guidelines
- `MEMORY.md` — proven episode structures and formats

## Process
1. Select a topic from the latest research output
2. Define the episode format: solo, interview, panel, or Q&A
3. Write 3 title options (each must include a specific outcome for the listener)
4. Write the episode description (2-3 sentences, use audience language)
5. Create the episode outline:
   - **Hook** (0:00-2:00): Why should the listener care? What will they get?
   - **Setup** (2:00-5:00): Context and background
   - **Main content** (5:00-25:00): 3-5 key points, each with examples
   - **CTA** (25:00-27:00): One clear action for the listener
   - **Close** (27:00-30:00): Summary and teaser for next episode
6. If interview format: write 5-7 questions and the "must-ask" question
7. Define the CTA: what action, where to track, expected conversion rate

## Outputs
- `outputs/YYYY-MM-DD_episode-plan_[topic-slug].md` — complete episode plan

## Quality Bar
- Title must pass the "would I click this?" test
- Outline must have a clear hook in the first 2 minutes
- Every main point must include at least one example or story prompt
- CTA must be specific and trackable (not "check out our website")
- Total planned length: 25-35 minutes

## Tools
- None required (pure content planning)

## Integration
- Receives input from RESEARCH skill
- Output goes to human for approval before production
- After episode airs, performance data feeds back into RESEARCH via weekly review
